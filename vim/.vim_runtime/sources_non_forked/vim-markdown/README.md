<h2>Installation</h2>

<p>If you use <a href="https://github.com/gmarik/vundle">Vundle</a>, add the following lines to your <code>~/.vimrc</code>:</p>

<pre class="chroma"><code><span class="nx">Plugin</span> <span class="s1">&#39;godlygeek/tabular&#39;</span><span class="err">
</span><span class="err"></span><span class="nx">Plugin</span> <span class="s1">&#39;plasticboy/vim-markdown&#39;</span></code></pre>

<p>The <code>tabular</code> plugin must come <em>before</em> <code>vim-markdown</code>.</p>

<p>Then run inside Vim:</p>

<pre class="chroma"><code><span class="p">:</span><span class="nx">so</span> <span class="p">~</span>/.<span class="nx">vimrc</span><span class="err">
</span><span class="err"></span><span class="p">:</span><span class="nx">PluginInstall</span></code></pre>

<p>If you use <a href="https://github.com/tpope/vim-pathogen">Pathogen</a>, do this:</p>

<pre class="chroma"><code><span class="nb">cd</span> ~/.vim/bundle
git clone https://github.com/plasticboy/vim-markdown.git</code></pre>

<p>To install without Pathogen using the Debian <a href="http://packages.qa.debian.org/v/vim-addon-manager.html">vim-addon-manager</a>, do this:</p>

<pre class="chroma"><code>git clone https://github.com/plasticboy/vim-markdown.git
<span class="nb">cd</span> vim-markdown
sudo make install
vim-addon-manager install markdown</code></pre>

<p>If you are not using any package manager, download the <a href="https://github.com/plasticboy/vim-markdown/archive/master.tar.gz">tarball</a> and do this:</p>

<pre class="chroma"><code><span class="nb">cd</span> ~/.vim
tar --strip<span class="o">=</span><span class="m">1</span> -zxf vim-markdown-master.tar.gz</code></pre>

<h2>Basic usage</h2>

<h3>Folding</h3>

<p>Folding is enabled for headers by default.</p>

<p>The following commands are useful to open and close folds:</p>

<ul>
<li><code>zr</code>: reduces fold level throughout the buffer</li>
<li><code>zR</code>: opens all folds</li>
<li><code>zm</code>: increases fold level throughout the buffer</li>
<li><code>zM</code>: folds everything all the way</li>
<li><code>za</code>: open a fold your cursor is on</li>
<li><code>zA</code>: open a fold your cursor is on recursively</li>
<li><code>zc</code>: close a fold your cursor is on</li>
<li><code>zC</code>: close a fold your cursor is on recursively</li>
</ul>

<p><a href="#options">Options</a> are available to disable folding or change folding style.</p>

<p>Try <code>:help fold-expr</code> and <code>:help fold-commands</code> for details.</p>

<h3>Concealing</h3>

<p>Concealing is set for some syntax such as bold, italic, code block and link.</p>

<p>Concealing lets you conceal text with other text. The actual source text is not modified. If you put your cursor on the concealed line, the conceal goes away.</p>

<p><a href="#options">Options</a> are available to disable or change concealing.</p>

<p>Try <code>:help concealcursor</code> and <code>:help conceallevel</code> for details.</p>

<h2>Options</h2>

<h3>Disable Folding</h3>

<ul>
<li><p><code>g:vim_markdown_folding_disabled</code></p>

<p>Add the following line to your <code>.vimrc</code> to disable the folding configuration:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_folding_disabled <span class="o">=</span> <span class="m">1</span></code></pre>
<p>This option only controls Vim Markdown specific folding configuration.</p>

<p>To enable/disable folding use Vim&rsquo;s standard folding configuration.</p>

<pre class="chroma"><code><span class="nb">set</span> <span class="o">[</span>no<span class="o">]</span>foldenable</code></pre></li>
</ul>

<h3>Change fold style</h3>

<ul>
<li><p><code>g:vim_markdown_folding_style_pythonic</code></p>

<p>To fold in a style like <a href="https://github.com/klen/python-mode">python-mode</a>, add the following to your <code>.vimrc</code>:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_folding_style_pythonic <span class="o">=</span> <span class="m">1</span></code></pre>
<p><code>g:vim_markdown_folding_level</code> setting (default 1) is set to <code>foldlevel</code>.
Thus level 1 heading which is served as a document title is expanded by default.</p></li>

<li><p><code>g:vim_markdown_override_foldtext</code></p>

<p>To prevent foldtext from being set add the following to your <code>.vimrc</code>:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_override_foldtext <span class="o">=</span> <span class="m">0</span></code></pre></li>
</ul>

<h3>Set header folding level</h3>

<ul>
<li><p><code>g:vim_markdown_folding_level</code></p>

<p>Folding level is a number between 1 and 6. By default, if not specified, it is set to 1.</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_folding_level <span class="o">=</span> <span class="m">6</span></code></pre>
<p>Tip: it can be changed on the fly with:</p>

<pre class="chroma"><code>:let g:vim_markdown_folding_level <span class="o">=</span> <span class="m">1</span>
:edit</code></pre></li>
</ul>

<h3>Disable Default Key Mappings</h3>

<ul>
<li><p><code>g:vim_markdown_no_default_key_mappings</code></p>

<p>Add the following line to your <code>.vimrc</code> to disable default key mappings:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_no_default_key_mappings <span class="o">=</span> <span class="m">1</span></code></pre>
<p>You can also map them by yourself with <code>&lt;Plug&gt;</code> mappings.</p></li>
</ul>

<h3>Enable TOC window auto-fit</h3>

<ul>
<li><p><code>g:vim_markdown_toc_autofit</code></p>

<p>Allow for the TOC window to auto-fit when it&rsquo;s possible for it to shrink.
It never increases its default size (half screen), it only shrinks.</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_toc_autofit <span class="o">=</span> <span class="m">1</span></code></pre></li>
</ul>

<h3>Text emphasis restriction to single-lines</h3>

<ul>
<li><p><code>g:vim_markdown_emphasis_multiline</code></p>

<p>By default text emphasis works across multiple lines until a closing token is found. However, it&rsquo;s possible to restrict text emphasis to a single line (i.e., for it to be applied a closing token must be found on the same line). To do so:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_emphasis_multiline <span class="o">=</span> <span class="m">0</span></code></pre></li>
</ul>

<h3>Syntax Concealing</h3>

<ul>
<li><p><code>g:vim_markdown_conceal</code></p>

<p>Concealing is set for some syntax.</p>

<p>For example, conceal <code>[link text](link url)</code> as just <code>link text</code>.
Also, <code>_italic_</code> and <code>*italic*</code> will conceal to just <em>italic</em>.
Similarly <code>__bold__</code>, <code>**bold**</code>, <code>___italic bold___</code>, and <code>***italic bold***</code>
will conceal to just <strong>bold</strong>, <strong>bold</strong>, <strong><em>italic bold</em></strong>, and <strong><em>italic bold</em></strong> respectively.</p>

<p>To enable conceal use Vim&rsquo;s standard conceal configuration.</p>

<pre class="chroma"><code><span class="nb">set</span> <span class="nv">conceallevel</span><span class="o">=</span><span class="m">2</span></code></pre>
<p>To disable conceal regardless of <code>conceallevel</code> setting, add the following to your <code>.vimrc</code>:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_conceal <span class="o">=</span> <span class="m">0</span></code></pre>
<p>To disable math conceal with LaTeX math syntax enabled, add the following to your <code>.vimrc</code>:</p>

<pre class="chroma"><code><span class="nb">let</span> g:tex_conceal <span class="o">=</span> <span class="s2">&#34;&#34;</span>
<span class="nb">let</span> g:vim_markdown_math <span class="o">=</span> <span class="m">1</span></code></pre></li>

<li><p><code>g:vim_markdown_conceal_code_blocks</code></p>

<p>Disabling conceal for code fences requires an additional setting:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_conceal_code_blocks <span class="o">=</span> <span class="m">0</span></code></pre></li>
</ul>

<h3>Fenced code block languages</h3>

<ul>
<li><p><code>g:vim_markdown_fenced_languages</code></p>

<p>You can use filetype name as fenced code block languages for syntax highlighting.
If you want to use different name from filetype, you can add it in your <code>.vimrc</code> like so:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_fenced_languages <span class="o">=</span> <span class="o">[</span><span class="s1">&#39;csharp=cs&#39;</span><span class="o">]</span></code></pre>
<p>This will cause the following to be highlighted using the <code>cs</code> filetype syntax.</p>

<pre class="chroma"><code><span class="sb">```</span>csharp
...
<span class="sb">```</span></code></pre>
<p>Default is <code>['c++=cpp', 'viml=vim', 'bash=sh', 'ini=dosini']</code>.</p></li>
</ul>

<h3>Follow named anchors</h3>

<ul>
<li><p><code>g:vim_markdown_follow_anchor</code></p>

<p>This feature allows the <code>ge</code> command to follow named anchors in links of the form
<code>file#anchor</code> or just <code>#anchor</code>, where file may omit the <code>.md</code> extension as
usual. Two variables control its operation:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_follow_anchor <span class="o">=</span> <span class="m">1</span></code></pre>
<p>This tells vim-markdown whether to attempt to follow a named anchor in a link or
not. When it is 1, and only if a link can be split in two parts by the pattern
&lsquo;#&rsquo;, then the first part is interpreted as the file and the second one as the
named anchor. This also includes urls of the form <code>#anchor</code>, for which the first
part is considered empty, meaning that the target file is the current one. After
the file is opened, the anchor will be searched.</p>

<p>Default is <code>0</code>.</p></li>

<li><p><code>g:vim_markdown_anchorexpr</code></p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_anchorexpr <span class="o">=</span> <span class="s2">&#34;&#39;&lt;&lt;&#39;.v:anchor.&#39;&gt;&gt;&#39;&#34;</span></code></pre>
<p>This expression will be evaluated substituting <code>v:anchor</code> with a quoted string
that contains the anchor to visit. The result of the evaluation will become the
real anchor to search in the target file. This is useful in order to convert
anchors of the form, say, <code>my-section-title</code> to searches of the form <code>My Section
Title</code> or <code>&lt;&lt;my-section-title&gt;&gt;</code>.</p>

<p>Default is <code>''</code>.</p></li>
</ul>

<h3>Adjust new list item indent</h3>

<ul>
<li><p><code>g:vim_markdown_new_list_item_indent</code></p>

<pre class="chroma"><code>* item1</code></pre>

<pre class="chroma"><code>* item1

<p>This is super useful for GitLab and GitHub wiki repositories.</p>

<p>Normal behaviour would be that vim-markup required you to do this <code>[link text](link-url.md)</code>, but this is not how the Gitlab and GitHub wiki repositories work. So this option adds some consistency between the two.</p></li>
</ul>

<p>In that case, you probably also want to set the new list item indent to 0 as
well, or you will have to remove an indent each time you add a new list item:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_new_list_item_indent <span class="o">=</span> <span class="m">0</span></code></pre></li>
</ul>

<h3>Change how to open new files</h3>

<ul>
<li><p><code>g:vim_markdown_edit_url_in</code></p>

<p>By default when following a link the target file will be opened in your current buffer.  This behavior can change if you prefer using splits or tabs by using the <code>vim_markdown_edit_url_in</code> variable.  Possible values are <code>tab</code>, <code>vsplit</code>, <code>hsplit</code>, <code>current</code> opening in a new tab, vertical split, horizontal split, and current buffer respectively.  Defaults to current buffer if not set:</p>

<pre class="chroma"><code><span class="nb">let</span> g:vim_markdown_edit_url_in <span class="o">=</span> <span class="s1">&#39;tab&#39;</span></code></pre></li>
</ul>

<h2>Mappings</h2>

<p>The following work on normal and visual modes:</p>

<ul>
<li><p><code>gx</code>: open the link under the cursor in the same browser as the standard <code>gx</code> command. <code>&lt;Plug&gt;Markdown_OpenUrlUnderCursor</code></p>

<p>The standard <code>gx</code> is extended by allowing you to put your cursor anywhere inside a link.</p>

<p>For example, all the following cursor positions will work:</p>

<pre class="chroma"><code><span class="o">[</span>Example<span class="o">](</span>http://example.com<span class="o">)</span>
^  ^    ^^   ^       ^
<span class="m">1</span>  <span class="m">2</span>    <span class="m">34</span>   <span class="m">5</span>       <span class="m">6</span>


&lt;http://example.com&gt;
^  ^               ^
<span class="m">1</span>  <span class="m">2</span>               <span class="m">3</span></code></pre>
<p>Known limitation: does not work for links that span multiple lines.</p></li>

<li><p><code>ge</code>: open the link under the cursor in Vim for editing. Useful for relative markdown links. <code>&lt;Plug&gt;Markdown_EditUrlUnderCursor</code></p>

<p>The rules for the cursor position are the same as the <code>gx</code> command.</p></li>

<li><p><code>]]</code>: go to next header. <code>&lt;Plug&gt;Markdown_MoveToNextHeader</code></p></li>

<li><p><code>[[</code>: go to previous header. Contrast with <code>]c</code>. <code>&lt;Plug&gt;Markdown_MoveToPreviousHeader</code></p></li>

<li><p><code>][</code>: go to next sibling header if any. <code>&lt;Plug&gt;Markdown_MoveToNextSiblingHeader</code></p></li>

<li><p><code>[]</code>: go to previous sibling header if any. <code>&lt;Plug&gt;Markdown_MoveToPreviousSiblingHeader</code></p></li>

<li><p><code>]c</code>: go to Current header. <code>&lt;Plug&gt;Markdown_MoveToCurHeader</code></p></li>

<li><p><code>]u</code>: go to parent header (Up). <code>&lt;Plug&gt;Markdown_MoveToParentHeader</code></p></li>
</ul>

<p>This plugin follows the recommended Vim plugin mapping interface, so to change the map <code>]u</code> to <code>asdf</code>, add to your <code>.vimrc</code>:</p>

<pre class="chroma"><code>map asdf &lt;Plug&gt;Markdown_MoveToParentHeader</code></pre>

<p>To disable a map use:</p>

<pre class="chroma"><code>map &lt;Plug&gt; &lt;Plug&gt;Markdown_MoveToParentHeader</code></pre>

<h2>Commands</h2>

<p>The following requires <code>:filetype plugin on</code>.</p>

<ul>
<li><p><code>:HeaderDecrease</code>:</p>

<p>Decrease level of all headers in buffer: <code>h2</code> to <code>h1</code>, <code>h3</code> to <code>h2</code>, etc.</p>

<p>If range is given, only operate in the range.</p>

<p>If an <code>h1</code> would be decreased, abort.</p>

<p>For simplicity of implementation, Setex headers are converted to Atx.</p></li>

<li><p><code>:HeaderIncrease</code>: Analogous to <code>:HeaderDecrease</code>, but increase levels instead.</p></li>

<li><p><code>:SetexToAtx</code>:</p>

<p>Convert all Setex style headers in buffer to Atx.</p>

<p>If a range is given, e.g. hit <code>:</code> from visual mode, only operate on the range.</p></li>

<li><p><code>:TableFormat</code>: Format the table under the cursor <a href="http://www.cirosantilli.com/markdown-style-guide/#tables">like this</a>.</p>

<p>Requires <a href="https://github.com/godlygeek/tabular">Tabular</a>.</p>

<p>The input table <em>must</em> already have a separator line as the second line of the table.
That line only needs to contain the correct pipes <code>|</code>, nothing else is required.</p></li>

<li><p><code>:Toc</code>: create a quickfix vertical window navigable table of contents with the headers.</p>

<p>Hit <code>&lt;Enter&gt;</code> on a line to jump to the corresponding line of the markdown file.</p></li>

<li><p><code>:Toch</code>: Same as <code>:Toc</code> but in an horizontal window.</p></li>

<li><p><code>:Toct</code>: Same as <code>:Toc</code> but in a new tab.</p></li>

<li><p><code>:Tocv</code>: Same as <code>:Toc</code> for symmetry with <code>:Toch</code> and <code>:Tocv</code>.</p></li>
</ul>
